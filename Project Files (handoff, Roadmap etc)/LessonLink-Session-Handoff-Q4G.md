## What Was Accomplished

### 1. PhonicsGame Review Mechanic

New self-contained `PhonicsGame` component in `unified-flashcard-review.tsx`. Works as follows:

- **Picture** of one word from a minimal pair (AI-generated via `generatePhonicsWordImage`)
- **Audio sentence** using the audio word (TTS via `speechSynthesis`, sentence generated by `generatePairSentence`)
- Picture word ≠ audio word 50% of the time (random match/mismatch)
- L clicks **✓ Match** or **✗ Mismatch** — scored immediately
- After answer: shows result, sentence text, picture vs audio word

**T-side preview:** Minimal pair chips are now clickable → opens PhonicsGame in a dialog for that specific pair. T can test before saving.

**L-side:** Never generates. Uses stored `gameData` from when T saved the card. Cards without `gameData` show "Card not ready" message.

---

### 2. gameData Stored with PhonicsCard

`PhonicsCard` type (in `types.ts`) now has optional `gameData` field:

```typescript
gameData?: {
  imageUrl: string;
  sentence: string;
  pictureWord: string;
  audioWord: string;
  otherWord: string;
  isMatch: boolean;
};
```

When T previews a pair and the `PhonicsGame` component generates image + sentence, `onGenerated` callback fires and stores data in `capturedGameData`. On save, `gameData` is written to the student subcollection card. L-side reads it directly.

---

### 3. Phonics Repository (`phonicsRepository` top-level collection)

Shared collection of phonics cards across all students.

**Flow when T selects a phoneme:**
1. App checks `phonicsRepository` for existing card (keyword + targetPhoneme match)
2. If found → shows green "✓ Found in repository" banner, pre-fills form + `capturedGameData`
3. If not found → generates fresh contrast via `generatePhonemeContrast`
4. "Generate new instead" button overrides repo card

**On save:**
- New card → adds to `phonicsRepository`
- Used repo card + T previewed a pair → patches `gameData` onto repo card if it was missing

**Browse:** T clicks "Browse" → loads all repo cards sorted by keyword in a scrollable list. No search field — just scroll. Each card has a "Use" button to pre-fill the form.

**Backfill:** One-time "Backfill" button scans all students' phonics subcollections and writes any missing cards to `phonicsRepository`. Run once to migrate existing cards.

**Firestore rules added:**
```
match /phonicsRepository/{cardId} {
  allow read: if isAuthenticated();
  allow create, update: if isAdmin() || isTeacher();
  allow delete: if isAdmin();
}
```

### 6. Grammar AI Prompt Fix

`generate-grammar-card.ts` improved prompt with chain-of-thought and explicit compatibility rules. Added examples:
- Present Perfect → use already/just/ever/never/yet/since/for — NOT yesterday/last week/ago

Also enforced single-error-only rule: one word wrong, rest of sentence correct.
